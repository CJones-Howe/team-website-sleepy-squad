Mitigation Tactics and Recommendations for Building In-Depth Defenses into IT Systems
1. Limit Sensitive Data Stored in the Systems
Organizations should apply the principle of data minimization, only collecting and retaining the data that is essential for business operations. This reduces the impact of a breach and minimizes the scope of data at risk. Key strategies include:
Classifying data based on sensitivity.
Automatically purging or archiving stale data.
Encrypting sensitive data at rest and in transit.
Applying strict access controls and user privileges.
2. Embed Security into Software Design and Development
Security must be integrated from the earliest stages of the Software Development Life Cycle (SDLC) â€” often called "shift-left" security. Recommendations include:
Conducting threat modeling during the design phase.
Enforcing secure coding practices and using code scanning tools (e.g., SAST/DAST).
Regularly performing code reviews and penetration testing.
Including security checks in CI/CD pipelines to catch vulnerabilities before deployment.
3. Protect Systems in Operation from Attacks
Operational systems must be hardened against threats through continuous protection and monitoring:
Patch management programs must ensure timely updates of software and libraries.
Network segmentation should be used to contain breaches.
Endpoint protection platforms (EPP) and firewalls must be configured correctly.
Implement multifactor authentication (MFA) and strict identity management controls.
4. Identify Attacks and Block Their Spread
Detection and containment are crucial for limiting the scope of a cyberattack:
Deploy intrusion detection systems (IDS) and Security Information and Event Management (SIEM) tools to detect anomalies.
Use behavior analytics to identify lateral movement or privilege escalation.
Define and rehearse incident response playbooks.
Employ network micro-segmentation and zero trust architecture to contain threats.
